Option Type

When we aren’t sure about the outcome of an operation then we can use the Option type. For example, we know that dividing by zero is undefined. But checking if a division is 0 and returning -1 as a signal that an undefined operation was tried, isn’t very clearn.

fn divide(div: i32, divisor: i32) -> i32 {
  if divisor == 0 {
    -1
  } else {
    div/divisor
  }

Instead, we can use the Option type. We do so by wrapping the return type in Option<T>, where T is the type of the returned value.

fn divide(div: i32, divisor: i32) -> Option<i32> {
  if divisor == 0 {
    // Rather than returning -1, if the case is invalid
    // we return None (which is basically the same as undefined)
    None
  } else {
    // If the case is valid, we simply state that the operation resulted in 
    // something and we return that something, as the value inside Some(T).
    Some(div/divisor)
  }
}

fn main(){
  let result = divide(10,2);
  // Using the Option type enable us to check and control flow depending on the
  // the result of the operation.
  if result.is_none() {
    println!("The operation resulted in undefined.", );
  } else {
    println!("{}", result.unwrap());
  }
}
}

Impl

We can use the Impl keyword to define methods for our structs. If we want a method to be aware of the instantiated object it operates on, we need to pass &self in the method signature.

impl Book {
  // We can call this method like so:
  // book.print_desc();
  fn print_desc(&self){
    println!("Title:{} // Price: {}", self.title, self.price);
  }
}
