ConcernsMotivation

There are a number of reasons to be careful when vibe coding on a VPS:

  1. Security: - the Ai might execute malicious code and delete valuable data on the server.
  2. YOLO: - continously accepting the AI’s suggested changes can get tedious, but auto-accepting changes (like Claud’s –dangerously-skip-permissions) is also risky.
  3. Permission: - some of the operations that the Ai want to use requires root access, which poses a risk.
  4. State Handling: - Claude Code is stateful which might be problematic if we concurrently access our server from different devices without resetting state.
  5. Sharing: - If multiple devs are working on the same project-claude state, that may cause an issue.
  6. Internet Access: - the Ai usually needs Internet access, so who provides that during long running processes? We want a Ai-coding environment which does not get interrupted due to Wifi disconnets.

Requirements

We want our Ai VPS setup to meet the following requirements:

  1. Hands-free: For a React webapp with a Python FastAPI backend, we want our AI to iterate without our commitment.
  2. Debug: - we want to be able to debug our webapp using Chrome with the Debug MCP repo.
  3. Persistence: - Our webapp’s backend needs to be able to talk to Supabase
  4. Claude: - able to run Claude Code.
  5. Cheap: - cheap is always nice.
  6. Easy to setup and use: - the Ai VPS needs to be easy to install, maintain, secure (Firewall).
  7. Secure: - we have to be able to trust our Ai VPS, given that we might be dealing with sensitive data.

Hardware

Based on our requirements, we can set the following hardware requirements:

  1. 2 Cores + 8 GB of ram with SSD as minimum hardware reqs.
  2. Low latency (< 50ms) to the area I live in on peak times (NYC Area)
  3. Less than $20 a month (you can buy a reasonable jump box for $120)
  4. Linux on x86 (some things still don‚Äôt run great on ARM)
  5. A provider that is considered stable and reliable.

The Build

Based on our list, we went with Hetzner. We picked a shared VPS server located in Nurmberb, Germany with 2 dedicated CPUs, 8GB of ram and 50GB of SSD storage, and with automated backups for $4 a month.

Backups are around $2 a month extra, and we’re using extra storage - which could also be scraped.

Process

These were the main steps:

  1. Set up account at www.hetzner.com
  2. Verify identity

Logging into the server can be a bit tricky, when it comes to SSH keys. Also beware, the VPS console can be extremely slow, but accessing from a loca machine via SSH has so far been fast.

NoMachine uses port 4000, so make sure to enable that incoming port on the Hetzner firewall

Setup and installations on the VPS

For good practice, do the following:

  • Create a personal account with sudo, rather than maintaining the machine using root.
  • Install the usuall Linux machine suspects: git, Xfce, etc.
  • Rather than installing using the Chrome Linux .deb, try going the repo route.
  • Make a shortcut to run Chome with the following params to enable your MCP to safely connect
        --remote-debugging-port=9222
        --remote-debugging-address=127.0.0.1
        --user-data-dir=/home/<user>/.chrome_dev_profile
  • Install the latest stable version of Node.js (try nodesource)
  • Install Python (sudo add-apt-repository ppa:deadsnakes/ppa)
  • Install uv (use pipx to install uv)
  • Install Claude Code (npm install -g @anthropic-ai/claude-code)
  • Install VSCode (add the Microsoft repo to apt)
  • Install the VSCode plugins for Claude Code
  • Install cline (with Claude Code selected for AI provider)
