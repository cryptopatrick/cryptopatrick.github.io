Comments

Commenting code is important. In Rust we have three ways of commenting code; single line comments, block comments, and doc commetns.

// This is a single line comment.
println!("Hello, world!");

/*
This is a multiline comment.
This is a multiline comment.
*/
println!("Hello, world!");

Constants

Constants are values which remain the same throughtout the running of the programs lifecycle. In constrast to let, there is no memory allocation in using const. Every use of a const value in the code will be replaced by its hard value during compile time. In contrast, using let allocates memory during runtime.

const PI: f64 = 3.14159;
// The difference between a const and using let is
// that there is no memory allocation in using  
fn main() {
  // Memory is allocated on the stack during runtime.
  let memory_pi: f64 = 3.14159
}

Variables

// We don't have to declare the type of a variable, as Rust can often infer it.
let score = 10;

// If we want, we can declare the type, just to be sure.
let highscore: i32 = 100;

// Variables are immutable by default, meaning that we have to declare them
// as `mut` in order to make the value changeable.
let mut new_highscore: i32 = 100

let bool is_it_true = false;

let char one_letter = 'a';

Enums

Enums in Rust are powerful, very powerful. The power of enums becomes evident when we use pattern matching as to control procedural flow. We can also have a variant containing its another type.

enum Seasons {
  Winter,
  Spring,
  // We want to express the average temp of the season.
  Summer(i32),
  Fall,
}

fn main() {
  let w = Seasons::Winter;
  let s = Seasons::Summer(30);

  match s {
    Season::Winter => println!("It's winter."),
    Season::Spring => println!("It's spring."),
    Season::Summer(value) => println!("It's summer and the temperatur is {}.", value),
    Season::Fall => println!("It's fall.")
  }
}

Loops

In Rust we have for loop, while loop, and the infinite loop.

for i in 1..10 {
  println!("{i}");
}

let guest_limit = 10;
let mut guests = 0;
while guests < guest_limit {
  guests += 1;
  println!("New guest arrived.");
}

// We can name the loop.
'run_forever: loop {

Match

fn main() {
    let age_man = 1;
    match age_man {
        1 => println!("one"),
        2 => println!("two"),
        n if n > 2 && n < 5 => println!("three or four"),
        _ => println!("Some other number"),
    }

    // Let's match return a value, with consistent type in all arms
    let num = "10";
    let age_woman: i32 = match num.parse::<i32>() {
        Ok(val) => val,
        Err(e) => {
            println!("There was an error in parsing: {}", e);
            0 // default value if parse fails
        }
    };

  println!("Parsed age: {}", age_woman);
  // We need to add this semicolon since match is returning a value 
  // (which makes this an expression.)
}

String and &str

User input

Getting input from a user of our program is essential. Let’s see how we can do so in Rust:

use std::io;

fn main(){
  // User instructions.
  println!("Enter the name of a calendar month:");
  // Read and store user input in our created variable.
  let mut input = String::new();
  io::stdin().read_line(&mut input);
  // Feedback.
  println!("You entered: {}", input);
}

Array Data Type

Arrays are of fixed size and can only contain one kind of data type. Indexes start from 0. Once created the size of an array cannot be changed. It’s important to understand that arrays of different are essentiall different data types, so, [i32;3] and [i32,5] are completely different types in Rusts perspective.

Arrays use stack memory.

fn main(){
  let _empty_chars: [bool;5];
  let pos_nums: [u32;3] = [1,2,4];
  println!("{}", pos_nums[1]); // prints 2
  // To print all elements of an array:
  for i in pos_nums.iter() {
    println!("{i}");
  }
  // We can also output the whole array as an object.
  println!("{:?}", pos_nums);
}

Vector Data Type

In contrast to arrays, vectors can change in size.

Vectors use Heap memory.

fn main(){
  // We use a handy vector creating macro.
  let ratings = vec![1,2,3];
  println!("{}",ratings.len());

  let mut movies: Vec<&str> = Vec::new();
  movies.push("Avatar");
  movies.push("Blade Runner");
  movies.push("Rambo");
  movies.pop(); // Removes last added item.
  movies.push("Cars");
  movies.remove(1); // Removes the 2nd index item (Blade Runner).
  movies.reverse(); // Removes the 2nd index item (Blade Runner).
  for m in movies.iter() {
    // Prints Cars and Avatar.
    println!("{m}");
  }
}

Closures

Imagine that we don’t care about nameing our function. A closure in Rust is basically an anonymous function - a function without a name.

// function signatures deman parameter types.
fn add_two_ints(x: u8, y: u8) -> u8 {
  return x + y;
}

fn main(){
  // In a closure Rust can infer parameter types.
  let x = 1;
  // closures have access outside their scope.
  // So the sum closure can captures the value of x.
  let sum = |y| x + y;
  println!("closure:{}", sum(1,2));
  println!("fn:{}", add_two_ints(10,30));
}

.iter, into_iter, iter_mut

  • .iter() : reference (read only copy) - when we don’t want to owned the value, just read it.
  • .iter_mut() : mutable reference - read and write, but we don’t own it, and it won’t be dropped when we leave the scope.
  • .into_iter() : owned value - we take over ownership and responsibility. Once exiting the scope of into_iter(){} the value will be dropped.
fn main(){
  let mut ratings = vec![1,2,3];
  for r in ratings.iter_mut() {
    *r = *r + 10;
    println!("Inside:{}", r);
  }
  println!("Outside:{:?}", ratings);
}
