Functions from build/instruction_template.rs are run to generate the source file instructions.rs, which is linked into the final binary. It’s responsible for generating the flattened Instruction data type as well as many boiler plate functions used during code generation.

Generation of the module is preferred because that way we can add new Instructions to the the template file and have them inserted into boiler plate functions automatically, eliminating the need to update them by hand. It’s so much more convenient.

2. Data Objects

A Prolog term is represented as a word containing a value and a tag. A large available adress space is assumed so that values can take up around 32 bitsin memory. The tag denotes the type of the term. It must be at least 2 bits and preferably up to 8 bits. The main types are:

  • references (to bound or unbound variables, with an unbound var pointing to itself)
  • structures
  • lists
  • constants (atoms and integers)

Structures and lists are represented by explicitly copying the functorand arguments into consecutive words or memory. Lists have a seprate tag from structures.

4. Registers and Treatment of Variables

A Prolog computation is defined by certain registers with pointers into the main data area. These are the main registers:

  • P : program pointer to the code area
  • CP : continuation program pointer to the code area
  • E : last environment on the local stack
  • B : last backtrack point on the local stack
  • A : top of the stack, not strictly essential
  • TR : top of trail
  • H : top of heap
  • HB : heap backtrack point , the H value corresponding to B
  • S : structure pointer to the heap
  • A1, A2, … : argument registers, used to pass the arguements to a procedure
  • X1, X2, … : used to hold values of a clause’s temporary variables

The A and X registers are identical. The naming is only there to reflect the different usages.

In the past, I’ve used SICStus Prolog and SWI-Prolog, but there’s a new Prolog on the proverbial “block” - Scryer Prolog.

What the hell is a Scryer?

# Download the binary (I'm on a Mac, but there are binaries for Win and Linux) 
curl -LO https://github.com/mthom/scryer-prolog/releases/download/v0.9.3/scryer-prolog_macos-11.zip
# Unzip the binary package and cd into it.
unzip scryer-prolog_macos-11.zip
cd scryer-prolog_macos-11_x86_64-apple-darwin

# Move the binary to the System directory /usr/local/bin.
mv scryer-prolog /usr/local/bin

# Make the binary an executable.
chmod +x /usr/local/bin/scryer-prolog

# Create an alias.
echo "alias prolog='scryer-prolog'" >> ~/.zshrc 
source ~/.zshrc

# Test that everything works.
prolog --version
prolog --help

MacOS: I got a security warning that prolog couldn’t be opened because the binary was download from the net. I solved this by going to System Preferences/Security & Privacy, and then clicking “Open anyway”.

Scryer-Prolog Configuration File

At startup, Scryer Prolog consults the file ~/.scryerrc, if the file exists. The config file is useful to automatically load libraries and define predicates that we’ll often need.

Beginner: The Scryer-Prolog README recommends the following: “Especially when starting with Prolog, we recommend to add the following directive to the ~/.scryerrc configuration file so that programming mistakes in predicates that lead to the creation of cyclic terms are indicated by errors:”

# Create a config file for our Scryer-Prolog. 
touch ~/.scryerrc
find ~/.scryerrc # Output: /Users/cp/.scryerrc
echo ":- set_prolog_flag(occurs_check, error)." >> ~/.scryerrc
echo ":- use_module(library(lists))." >> ~/.scryerrc
echo ":- use_module(library(dcgs))." >> ~/.scryerrc
echo ":- use_module(library(reif))." >> ~/.scryerrc

Start a session

Prolog files are ‘consulted’, which means loading them into a Prolog session. For example, to load program.pl, use:

$> scryer-prolog program.pl

Quit: To quit a Scryer Prolog session, use the standard predicate halt/0: ?- halt..

Predicate Arrity: in Prolog, what we’d call a function in most imperative programming languages, is called a predicate. Each predicate is denoted by its name and the number of arguements it takes, e.g. hello/1 takes one arguement.

Once in the shell we can use the built-in predicate consult/1 to consult a file:

?- consult('program.pl').
