When we code in Rust, it’s nice to think that we’re close to the metal (CPU). In reality, there’s a whole abstraction layer lodged between our Rust code and the CPU - that layer is Assembler. Assembly code are the actual machine instructions to the CPU. We can’t get any lower, or close to the metal (CPU), than Assembler code. As such, it makes sense to have a good understanding what our Rust code is actually instructing, via generated Assembler code, the CPU to do.

Increment integer by 1

pub fn increment_i64(num: i64) -> i64 {
  num + 1
}

// lea : load effective address adds 1 to the value in ``rdi``.
lea rax, [rdi + 1]
// return the value stored in rax as a confimation.
ret
