Algorithmic Information Theory

Recent discoveries have unified the fields of computer science and information theory into a new field: algorithmic information theory (AIT). This field is also known by its central result, Kolmogorov complexity.

Kolmogorov complexity gives us a new way to understand the mathematics of information, which describes the structures of the world‚Äîranging from cultural systems (science, law, markets, art, and beliefs) to biological and physical phenomena. Its most direct applications are in computing and communications.

This essay provides an overview of AIT. Only a basic understanding of computer science and probability theory is required.

Describing Objects with Binary Strings

We describe objects using binary strings. The length of the shortest string that describes an object is a measure of that object‚Äôs complexity.

In traditional information theory, communication is described as follows:

  • The sender and receiver share a specification method L.
  • Message x can be transmitted as y, such that L(y) = x.

This is written:

y : L(y) = x

The cost of transmission is the length of y, denoted |y|. The least cost is:

min(|y|) : L(y) = x

This minimal |y| is the descriptional complexity of x under L.

A Universal Description Method

A universal description method should:

  • Be independent of the particular L, within a constant offset, so complexities can be compared across all objects.
  • Be performable by either machines or humans.

Such a method measures absolute information content‚Äîthe amount of data needed to describe an object in the absence of any prior knowledge.

Kolmogorov Complexity

The description method that satisfies these criteria is Kolmogorov complexity: the size of the shortest program (in bits) that, without additional input, outputs a given string and halts.

Formally:

K(x) = min |p| : U(p) = x

where U is a universal Turing machine.

Variations on Kolmogorov Complexity

  • Conditional Kolmogorov complexity:

    K(x | y) = min |p| : U(p, y) = x

  • Joint Kolmogorov complexity:

    K(x, y) = min |p| : U(p) = xy

Examples

  1. œÄ (Pi) Though œÄ is an infinite sequence of digits, it can be generated by a short program. Thus, its descriptional complexity is constant:

    K(œÄ) = O(1)

  2. Random String A truly random string is incompressible‚Äîits description length is about its total length:

    K(x) = Œò(|x|)

Algorithmic Probability and Induction

In a long series of coin flips, regular sequences are highly improbable, while irregular ones dominate. Wherever we see regularity, we infer causation‚Äîbecause compressibility implies causation.

We redefine probability using computational theory via Kolmogorov complexity:

P(p) = 2^(|x| - K(x))

This represents the chance that x was generated by a short program rather than by randomness.

Variations on Algorithmic Probability

When allowing for error (or distortion) between x and an approximation y:

P(p) = 2^(|x| - D(x, y) - K(x))

The universal discrete semimeasure (probability that a random program outputs x) is:

m(x) = Œ£ (over all p : U(p) = x) 2^(-|p|) + c

The algorithmic probability of x is:

R(x) = 2^(-K(x))

Coding Theorems

K(x) = Œò(-log m(x))

Conditional coding theorem:

K(x | y) + Œò(1) = -log m(x | y)

Prediction

Rudolf Carnap proposed predicting by taking a weighted sum of all explanations in a given language. Ray Solomonoff generalized this to the universal language of Turing machines:

Predict by taking a weighted combination of all programs that explain the data and observe what they print next.

Occam‚Äôs Razor

‚ÄúIt is vain to do with more what can be done with fewer.‚Äù

Kolmogorov complexity provides an objective measure of simplicity. The Minimum Description Length (MDL) principle states:

Minimize the weighted sum of the model‚Äôs complexity and its error with respect to the data.

Distance

To formalize similarity between two objects (e.g., images represented as bitstrings), Li and Vit√°nyi introduced information distance:

E(x, y) = max(K(y | x), K(x | y))

This measures the shortest program that can transform x into y and vice versa.

The irreversible information distance is:

KR(x, y) = K(y | x) + K(x | y)

KR represents the minimal work required to transform x into y when intermediate results are not retained.

Superficiality and Sophistication

Charles Bennett proposed logical depth as a measure of sophistication.

  • Superficial objects (e.g., a garbage pit) can be created by many random processes.
  • Sophisticated objects (e.g., a functioning airplane or biological eye) have high replacement cost‚Äîthey require structured, adaptive algorithms to reproduce.

Formally, logical depth is:

The time required by a universal Turing machine to compute an object from its shortest description.

Depth thus represents the computational replacement cost of an object and can even serve as an objective measure of value.

Part and Whole

Compression can be computationally infeasible for large strings, and compressing parts may depend on the whole. Key questions include:

  • Is it easier to compress parts first or the whole at once?
  • Can patterns found in one part help compress others?

These questions mirror the part‚Äìwhole problems in hermeneutics and remain one of the last challenges for a consistent, contradiction-free theory of induction.

Total and Partial Order

Binary strings represent a total order of bits. A more general theory would handle partial orders, relevant when event orderings are incomplete (e.g., in physical observations).

Computable Description Methods

In general, K(x) is uncomputable‚Äîwe cannot always determine whether a program halts. However, we can derive computable entropies (descriptional complexities) for restricted computation structures (e.g., polynomials, decision trees, finite automata).

These serve as objective functions for learning algorithms‚Äîsuch as fitness functions in genetic programming.

In general:

H = structural entropy + remaining sample entropy

Examples:

  • Polynomial fitting (degree k, precision d):

    H(k, d) = kd + Œò(log kd) + Œ£_i s * (f(x_i) - y_i)^2

  • Decision trees:

    H = (# of nodes) + (# of bits in inconsistent examples)

  • Game strategies: The number of states in a finite-state machine describes bounded rationality. A more precise measure would include time, space, and learnability costs.

Conclusion

Algorithmic Information Theory is a far-reaching synthesis of computer science and information theory. Its applications extend well beyond computation‚Äîto mathematics, induction, hermeneutics, and the philosophy of knowledge itself.

% Predicate to simulate a universal Turing machine U(p) = x
% Simplified: program p is a string, output x is a string, and |p| is its length
turing_machine(Program, Output) :-
    % Example: Program is a prefix of Output (simplified model)
    string_length(Program, ProgLen),
    string_length(Output, OutLen),
    ProgLen =< OutLen,
    sub_string(Output, 0, ProgLen, _, Program).

% Kolmogorov complexity K(x) = min |p| : U(p) = x
kolmogorov_complexity(X, K) :-
    % Find shortest program that outputs X
    findall(ProgLen, (turing_machine(Program, X), string_length(Program, ProgLen)), Lengths),
    Lengths \= [], % Ensure there is at least one program
    min_list(Lengths, K).

% Conditional Kolmogorov complexity K(x|y) = min |p| : U(p, y) = x
conditional_kolmogorov_complexity(X, Y, K) :-
    % Simplified: Program uses Y as input, outputs X
    string_concat(Y, P, Program), % Program P uses Y
    turing_machine(Program, X),
    string_length(P, K),
    findall(ProgLen, (string_concat(Y, P2, Prog2), turing_machine(Prog2, X), string_length(P2, ProgLen)), Lengths),
    min_list(Lengths, K).

% Joint Kolmogorov complexity K(x, y) = min |p| : U(p) = xy
joint_kolmogorov_complexity(X, Y, K) :-
    string_concat(X, Y, XY), % Concatenate X and Y
    kolmogorov_complexity(XY, K).

% Algorithmic probability R(x) = 2^(-K(x))
algorithmic_probability(X, Prob) :-
    kolmogorov_complexity(X, K),
    Prob is 2 ** (-K).

% Universal discrete semimeasure m(x) = Œ£ (over all p : U(p) = x) 2^(-|p|)
universal_semimeasure(X, M) :-
    findall(Prob, (turing_machine(Program, X), string_length(Program, Len), Prob is 2 ** (-Len)), Probs),
    sum_list(Probs, M).

% Information distance E(x, y) = max(K(y|x), K(x|y))
information_distance(X, Y, E) :-
    conditional_kolmogorov_complexity(Y, X, KY_X),
    conditional_kolmogorov_complexity(X, Y, KX_Y),
    E is max(KY_X, KX_Y).

% Irreversible information distance KR(x, y) = K(y|x) + K(x|y)
irreversible_information_distance(X, Y, KR) :-
    conditional_kolmogorov_complexity(Y, X, KY_X),
    conditional_kolmogorov_complexity(X, Y, KX_Y),
    KR is KY_X + KX_Y.

% Logical depth: Simplified as the "time" to compute X from its shortest program
logical_depth(X, Depth) :-
    kolmogorov_complexity(X, K),
    % Simplified: Depth is proportional to K (e.g., number of steps to execute)
    Depth is K * 10. % Arbitrary scaling factor for simulation

% Example: Structural entropy for polynomial fitting H(k, d) = kd + Œò(log kd) + error term
polynomial_fitting_entropy(K, D, DataPoints, H) :-
    % K: degree, D: precision, DataPoints: list of (x_i, y_i)
    length(DataPoints, N),
    LogTerm is log(K * D) / log(2), % Convert to base 2
    % Simplified error term: sum of squared differences (assumes f(x) = x^K for simplicity)
    findall(Error, (member((X, Y), DataPoints), FX is X ** K, Error is (FX - Y) ** 2), Errors),
    sum_list(Errors, ErrorSum),
    H is K * D + LogTerm + ErrorSum.
