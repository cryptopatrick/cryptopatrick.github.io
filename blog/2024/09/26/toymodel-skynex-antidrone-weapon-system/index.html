ToyModel: Skynex AntiDrone Weapon System

September 26, 2024 by CryptoPatrick Mathematics Modelling Programming

I read a conversation on Twitter regarding a new weapon system from german company Reihmetal. The system is called Skynex. Just out of curiousity, but also to practice using the math I’m learning to model stuff to understand how things might fit together.

Model Components and Variables

1.Search Radar Detection (10 km range)

  • Drones are detected at a distance of up to $R_\text{search} = 10km$.
  • The system can track and prioritize multiple drones simultaneously.
  1. Engagement Range (4 km)
  • The gun system engages drones when they come within $R_\text{engage} = 4km$.
  1. Number of Cannons in a Battery
  • For $50$ drones, at least $N_\text{cannons} = 7$.
  1. Gun Performance
  • The cannon can intercept up to $N_\text{drones} = 8$ small drones within a few seconds.
  • Each cannon has a capacity of $C_\text{rounds} = 250$ rounds
  • The system uses proximity ammo, reducing the need for direct hits. Let’s assume $N_\text{rounds} = 5$ rounds are used per drone on average.
  1. Drone Characteristics
  • Drones travel at speed $v_\text{drone}$ in $m/s$ which will vary depending on the type of drone.
  • There could be up to $50$ drones approaching, with a spacing of $d_\text{drone}$ between them (either as a swarm or spread out).
  1. Radar and Sensor Performance
  • The system employs radar, TV-tracking, IR-tracking, and a laser range finder, making jamming or evasion by drones extremely unlikely.
  • Let’s assume the radar can handle simultaneous tracking of $N_\text{radar} = 100$ drones.
  1. Engagement Timing
  • Engagement is automatic, based on prioritization by speed and course.
  • Reaction time $t_\text{reaction}$ between detection and engagement is determined by sensor processing and mechanical limitations of the gun.

Equations

Time to Engagement

The time for a drone to move from radar detection range to engagement range is given by: $$t_\text{engage} = \frac{R_{search} - R_{engage}}{v_{drone}}$$

Firing Time per Drone

If we denote Where $r_\text{fire}$ as the cannon’s firing rate in rounds per second, and we assume that each cannon engages $N_\text{drones} = 8$ drones simultaneously within a few seconds, the time to destroy $N_\text{drones}$ is: $$t_\text{firing} = \frac{N_{rounds} - N_{drones}}{r_{fire}}$$

Rounds Spent Per Target

On average, the system uses $N_\text{rounds} = 5$ rounds per drone, meaning that for $N_\text{targets}$ drones, the total number of rounds spent is: $$R_\text{total} = N_{rounds} * N_{targets}$$

Drone Saturation

If $P_\text{saturation} > 1$, then the system is considered saturated (meaning drones will have successfully penetrated their targeted defence perimiter).

The system can handle $N_\text{radar}$ drones being tracked at once. The probability $P_\text{{saturation}$ that the system becomes saturated dedepnds on the number of incoming drones: $$P_\text{saturation} = \frac{N_{drones}}{N_{radar}}$$

Cannon Rotation and Reaction Time

The system’s reaction time matters. We’ll denote it as $t_\text{reaction}$, representing the time it takes for the cannon+radar system to detect, prioritize, aim and fire at the next set of drones. This variable can be approximated by a function of sensor processing speed s_process and mechanical movment (rotating the gun turret) m_rotate. We assume the reaction time of the system is a constant t_reaction = 2 seconds. This gives us: $$t_\text{reaction} = f(s_{process}, m_{rotate})$$
