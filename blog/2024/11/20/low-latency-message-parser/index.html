In today’s Everyday I’m exploring the bare bones of creating a message parser and feed handler for direct financial market data feeds. A reminder that this year’s Everyday theme is using: Rust, React, and SQLite.

Step 1: Project Setup

cargo new market_data_feed_parser
cd market_data_feed_parser

Add dependencies to Cargo.toml so that we can do parsing, benchmarking, and create macros:

cargo add nom
cargo add serde
cargo add criterion

Step 2: Create Type System for Market Data

We use Rust powerful enums to create a type system that models the type of mesages that we expect to parse:

use serde::{Serialization, Deserialization};

////////////////////////////////////////////////////////////////////////////////
/// Financial Market Data Type System
/// Enum representing an individual order in an order book.
#[derive(Debug, Serialize, Deserialize)]
pub enum Order {
  pub price: f64,
  pub volume: f64,
}

/// Enum representing the entire order book for a particular market instrument.
#[derive(Debug, Serialize, Deserialize)]
pub enum OrderBook {
  pub symbol: String, // "GBPUSD", "BTCUSD",
  pub bids: Vec<Order>,
  pub asks: Vec<Order>,
}

/// Enum representing trading report messages.
#[derive(Debug, Serialize, Deserialize)]
pub enum TradeReport {
  pub trade_id: u64,
  pub symbol: String, // "GBPUSD", "BTCUSD",
  pub price: f64,
  pub volume: u64,
  pub timestamp: u64, // Timestamp in nanoseconds.
}

/// Enum representing an individual price update.
#[derive(Debug, Serialize, Deserialize)]
pub enum PriceUpdate {
  pub symbol: String,
  pub price: f64,
  pub timestamp: u64, // Timestamp in nanoseconds.
}

/// Enum representing the different types of messages we can handle.
#[derive(Debug, Serialize, Deserialize)]
pub enum FeedMessage {
  PriceUdate(PriceUpdate),
  TradeReport(TradeReport),
  OrderBook(OrderBook),
}
