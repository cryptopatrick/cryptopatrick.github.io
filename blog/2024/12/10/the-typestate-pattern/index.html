The Art of casting Spells

When learning a new concept I often try to come up with an imaginary example. I feel that it helps me remember things, and generally makes learning more fun. Today, the concept is TypeState and I’m imagining a wizard casting spells.

In the classic Role-Playing Game, Dungeons & Dragons (or D&D RPG), magic is a powerful force.
In order to create a balanced game, there has to be limits to the use of magic.
The rules of D&D states that:

  • It takes Magical Energy to cast spells.
  • A wizard has limited magical energy and needs to “recharge” once it has been spent
  • The way a wizard can “recharge” magical energy is with a full nights sleep

Also, there has to be a ranking system of spells, from lesser to more powerful, where there is a greater “cost” in magical energy to casting a 5th level spell compared to a 1st level (weaker) spell.

State Machine: Magic Energy

We can take the the constraints placed by the D&D rules on Wizards and their access to magic energy, and express it as a state machine:

  1. Rested
    “If magic energy is not exhausted” -> “Can cast spells”
    “Else if magic energy is exhausted” => Tired
  2. Tired
    “If has no magic energy to cast spells with” -> “Cannot cast spells”
    “If go to sleep” => Sleep
  3. Sleep
    “If hasn’t slept a full nights sleep” -> “Cannot cast spells”
    “If has slept a full nights sleep” => Rested

The TypeState Pattern

I’m not a D&D player mayself but let’s assume our state machine is correct. So, next, let’s use Rust’s mighty type system and the TypeState pattern to express the constraints previously laid out.
Given a struct Wizard, our goal is:

  1. Depending on the state of the Wizard, some methods should be callable.
  2. Depending on the state of the Wizard, some methods should not be callable.
  3. We want to declare how a wizard can move from one state to another.

Okay, neat - but how is the TypeState pattern actually implemented?
Good question. Let’s look at some code and see how we can go about implementing this pattern.

// File: main.rs
use std::marker;

////////////////////////////////////////////////////////////////////////////////
// TypeStates
//
// We express the different types of `State` that our Wizard can move between.
// Each state is its own struct.
struct Rested;
struct Tired;
struct Sleeping;

// An enum is used to enable us to return multiple state paths from one state.
enum EnergyCheck {
    CanCastAgain(Wizard<Rested>),
    NeedsRest(Wizard<Tired>),
}

// Our wizard is expressed here.
struct Wizard<State> {
    // Every Wizard needs a name.
    name: String,
    // As the wizard casts spells its energy will go down.
    energy: u8,
    // Once the wizard is rested its energy will be set to rested_energy.
    rested_energy: u8,
    // The _state field is of type std::marker::PhantomData<State>.
    // This is a marker type provided by Rust, which enable us to associate
    // the TypeState without actually storing a value of type.
    // 
    // Since we're not goint to use the field for anything other
    // than to carry state, by Rust convention, the field is prefixed 
    // with an underscore.
    _state: marker::PhantomData<State>,
}

impl Wizard<Rested> {
    // When we create a new wizard, that wizard starts off in a Rested state.
    // Please note that this is the only constructor for the Wizard struct.
    fn new(name: String, energy: u8) -> Wizard<Rested> {
        println!(
            "Creating a new Wizard, named: {}, with energy: {}.",
            name, energy
        );
        Wizard {
            name: name,
            energy: energy,
            rested_energy: energy,
            _state: marker::PhantomData,
        }
    }
}

impl Wizard<Rested> {
    fn cast_spell(mut self) -> EnergyCheck {
        // Every time the wizard casts a spell we reduce its magic_energy.
        // If the energy is reduced to zero, the wizard becomes tired and
        // cannot cast any spells until rested.
        match self.energy > 0 {
            true => {
                self.energy = self.energy - 1;
                println!(
                    "The rested Wizard, {}, cast a spell and now
                    has {} magic energy points left.",
                    self.name, self.energy
                );
                return EnergyCheck::CanCastAgain(self);
            }
            false => {
                println!(
                    "The Wizard, {}, has exhausted its magic energy and
                    has to sleep to restore its magic energy.",
                    self.name
                );

                EnergyCheck::NeedsRest(Wizard::<Tired> {
                    name: self.name,
                    energy: self.energy,
                    rested_energy: self.rested_energy,
                    _state: marker::PhantomData,
                })
            }
        }
    }
}

// The only available method for Tired Wizards is to go to sleep.
// There is no way for a Tired Wizard to cast a spell because there
// is not such method to call.
impl Wizard<Tired> {
    fn go_to_sleep(self) -> Wizard<Sleeping> {
        println!("The tired Wizard, {}, went to sleep.", self.name);
        Wizard {
            name: self.name,
            energy: self.energy,
            rested_energy: self.rested_energy,
            _state: marker::PhantomData,
        }
    }
}

// The only available method for Tired Wizards is to wak up.
// There is no way for a Sleeping Wizard to cast a spell because there
// is not such method to call.
impl Wizard<Sleeping> {
    fn wakeup(self) -> Wizard<Rested> {
        println!("The Wizard, {}, is now fully rested.", self.name);
        Wizard {
            name: self.name,
            energy: self.rested_energy,
            rested_energy: self.rested_energy,
            _state: marker::PhantomData,
        }
    }
}

fn main() {
    let gandalf = Wizard::new(String::from("Gandalf"), 2);
    let gandalf = gandalf.cast_spell(); // gandalf is now of type EnergyCheck.
    // Fails because EnergyCheck does not have a method called .cast_spell()
    let gandalf = gandalf.cast_spell();
}
