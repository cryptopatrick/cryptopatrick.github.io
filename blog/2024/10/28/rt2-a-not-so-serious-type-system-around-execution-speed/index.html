RT2 is a type system that enable us to express types around enforced execution time guarantees using a type system, preventing misuse of operations that exceed specified execution times. In this post I’ll go through the idea, the code, and also sketch a few examples of how the code could be used in the military domain, where timing is critical. Creating a type system in Rust that classifies operations based on their execution timeâ€”namely Fast, Medium, and Slowâ€”requires us to leverage Rust’s type system and some creative use of traits. Below is an implementation that meets your requirements, complete with documentation, tests, and three examples from the military domain.

// Define a trait for execution time
pub trait ExecutionTime {
    fn duration(&self) -> u128; // Duration in milliseconds
}

// Marker types for Fast, Medium, and Slow
pub struct Fast<T>(T);
pub struct Medium<T>(T);
pub struct Slow<T>(T);

// Implementation for Fast type
impl<T: ExecutionTime> ExecutionTime for Fast<T> {
    fn duration(&self) -> u128 {
        self.0.duration()
    }
}

// Implementation for Medium type
impl<T: ExecutionTime> ExecutionTime for Medium<T> {
    fn duration(&self) -> u128 {
        self.0.duration()
    }
}

// Implementation for Slow type
impl<T: ExecutionTime> ExecutionTime for Slow<T> {
    fn duration(&self) -> u128 {
        self.0.duration()
    }
}

// Operations that are guaranteed to finish in less than 1 millisecond
impl<T> Fast<T> {
    pub fn new(value: T) -> Self {
        Fast(value)
    }
}

// Operations that are guaranteed to finish in less than 100 milliseconds
impl<T> Medium<T> {
    pub fn new(value: T) -> Self {
        Medium(value)
    }
}

// Operations that cannot guarantee to finish in less than 100 milliseconds
impl<T> Slow<T> {
    pub fn new(value: T) -> Self {
        Slow(value)
    }
}
